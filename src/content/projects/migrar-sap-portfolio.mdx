## Qué es

Migrar SAP Portfolio resume una automatización real de cuentas a pagar llevada a una versión pública segura. El objetivo no es mostrar solo macros o scripts sueltos, sino un flujo completo donde documentos, datos, reglas y acciones operativas quedan conectados.

## Problema que resuelve

El caso parte de un escenario típico de AP: alto volumen, múltiples validaciones, cruce entre sistemas y necesidad de sostener productividad sin perder control.

El reto técnico y funcional es transformar ese circuito en un flujo más confiable, repetible y menos manual.

## Solución implementada

La solución combina Excel y VBA como base operativa con apoyo de Python, Power Query, automatización web y scripting sobre SAP GUI. La versión pública conserva arquitectura, reglas determinísticas y evidencia reproducible, sin exponer datos sensibles ni integraciones corporativas reales.

## Stack principal

- Excel + VBA
- Python
- Power Query
- SAP GUI scripting
- Selenium
- tooling de testing y export para versión pública

## Flujo end-to-end

- ingreso y organización de documentos PDF
- extracción y normalización de datos
- cruce con reportes y sistemas internos
- aplicación de validaciones fiscales y de negocio
- generación de salidas operativas y evidencia reproducible

## Qué demuestra

- automatización end-to-end con impacto operativo real
- integración entre sistemas, archivos y reglas de negocio
- diseño orientado a trazabilidad, validación y reducción de errores
- criterio para publicar un caso sensible en formato portfolio-safe

## Estado actual

Al 15 de marzo de 2026, el repositorio presenta una release pública enfocada en arquitectura, reglas y evidencia. No expone el entorno corporativo completo, pero sí deja visible la lógica, la estructura modular y el valor técnico del caso.

## Valor profesional

Este proyecto muestra la parte menos visible pero más diferencial de mi perfil: entender procesos complejos, convertirlos en automatizaciones sostenibles y balancear contexto funcional con ejecución técnica.
